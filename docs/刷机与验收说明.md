# 刷机与验收说明

## 刷机前

1. 确认设备是京东云雅典娜 AX6600，型号 RE-CS-02。
2. 备份当前配置，但跨分支刷机不要恢复整包配置。
3. 备份 ART/校准等重要 MTD 分区。
4. 准备可用的 U-Boot 或 TTL 救砖路径。
5. 核对文件名必须包含 `jdcloud_re-cs-02`。
6. 从现有 OpenWrt/LibWrt 升级时使用 `sysupgrade.bin`，不使用 `factory.bin`。
7. 取消“保留配置”。

## 刷完后的验收顺序

### 一、只接电脑与雅典娜 LAN

访问：

```text
http://192.168.50.1
```

设置 root 密码。

### 二、检查系统与 BTF

将 `verify-after-flash.sh` 上传到 `/tmp`，执行：

```sh
chmod +x /tmp/verify-after-flash.sh
/tmp/verify-after-flash.sh
```

必须确认：

- `/sys/kernel/btf/vmlinux` 存在；
- `dae --version` 有输出；
- `kmod-sched-bpf`、`kmod-veth` 已安装；
- LuCI 有“服务 → daede”。

### 三、先不启用 DAE，完成 WAN 联网

确认：

```sh
ping -c 3 1.1.1
nslookup github.com
```

均正常。

### 四、关闭冲突组件

不要同时启用：

- PassWall/PassWall2
- OpenClash
- HomeProxy
- 其他 TPROXY/TUN 透明代理
- NSS ECM（初次调试阶段）

### 五、DAE 最小化测试

1. 后端选择 `dae`。
2. LAN 接口为 `br-lan`。
3. WAN 接口为 `auto`。
4. 只导入一个已验证可用的节点。
5. 先不开复杂 DNS、负载均衡和多节点策略。
6. 启动后观察日志，再测试国内外网站。

### 六、出现断网时

立即停 DAE：

```sh
/etc/init.d/dae stop
```

查看：

```sh
logread | grep -Ei 'dae|bpf|btf|xdp|veth|failed|error' | tail -n 200
```

不要使用 `opkg --force-depends` 安装其他固件的 kmod。
